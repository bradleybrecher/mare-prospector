# `backend/secrets/`

Service account JSON keys for Google Cloud APIs (Vertex AI, Google Photos, etc.)
live here. Everything in this folder **except this README** is gitignored.

## Expected files

| File | Purpose | How to obtain |
| --- | --- | --- |
| `vertex-sa.json` | Vertex AI service account key. Referenced by `GOOGLE_APPLICATION_CREDENTIALS` in `.env`. | [Cloud Console → IAM → Service Accounts](https://console.cloud.google.com/iam-admin/serviceaccounts?project=mare-head-spa) → create `mare-vertex` with **Vertex AI User** role → Keys tab → Add Key → JSON. |
| `google_photos_oauth_client.json` | OAuth 2.0 desktop client for Google Photos uploads. | [Cloud Console → APIs → Credentials](https://console.cloud.google.com/apis/credentials?project=mare-head-spa) → Create credentials → OAuth client ID → Application type: **Desktop app**. |
| `google_photos_token.json` | Created automatically after first `mare publish photos-auth` run. Contains the user's refresh token. | Generated by the app — do not create by hand. |

## Rotation

If a key leaks, disable it in the Cloud Console **first**, then delete the
local file. Generate a fresh one and replace.

## On servers / CI

Don't commit these. Mount them at runtime via the deployment's secrets
manager (AWS Secrets Manager, GCP Secret Manager, GitHub Actions secrets,
etc.) and point `GOOGLE_APPLICATION_CREDENTIALS` at the mount path.
